Addressing poor backend architecture

The site was completely re-architected paying focus on improving these:

  • Easy to manage multi-language content
  • Intuitive categorisation of content based on content categories on the frontend
  • Easy to manage content with guidance on allowed media types, dimensions, file sizes so a non-technical admin can easily manage the entire site
  • Easy to manage search engine content like page url name, page meta details, URL redirects, graceful 404 pages
  • The minimum click theory is also followed on the backend, so the admin can get his intended action done in the least amount of clicks
